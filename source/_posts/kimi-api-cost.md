## 费用明细

一周的 API 消耗统计：

| 服务 | 用途 | 调用次数 | Token 消耗 | 费用（约） |
|------|------|---------|-----------|-----------|
| Claude API | 内容生成 | 89 次 | 1,240K tokens | ¥62 |
| Kimi API | 长文总结 | 34 次 | 680K tokens | ¥18 |
| OpenAI API | 图片理解 | 12 次 | — | ¥8 |
| 飞书 API | 数据读取 | 156 次 | — | 免费 |
| **总计** | | **291 次** | | **¥88** |

一周 88 块。一个月大约 **350 块**。

fifi 的反应：

> "一个月 350……比我的视频会员都贵。"

## 费用大头分析

88 块里，**Claude 占了 70%**。为什么？

1. **Prompt 太长** —— 每次调用都带了大量系统提示和上下文
2. **重复调用** —— 同一篇文章改了 3-4 次，每次都是全量调用
3. **没有缓存** —— 相似的请求没有复用结果

## 优化方案

### 1. 压缩 Prompt

```
优化前：系统提示 2000 tokens + 上下文 3000 tokens = 5000 tokens/次
优化后：系统提示 800 tokens + 精简上下文 1200 tokens = 2000 tokens/次
节省：60%
```

### 2. 引入缓存层

对于相似请求，先查缓存再调 API：

```javascript
const cacheKey = hash(prompt.slice(0, 200));
const cached = await cache.get(cacheKey);
if (cached && Date.now() - cached.time < 3600000) {
  return cached.result;
}
```

### 3. 分级使用

| 任务类型 | 推荐模型 | 原因 |
|---------|---------|------|
| 简单格式转换 | Kimi | 便宜，够用 |
| 内容生成 | Claude | 质量最好 |
| 图片理解 | GPT-4V | 独家能力 |
| 数据整理 | Kimi | 性价比高 |

**核心原则**：不是所有任务都需要最贵的模型。

## 优化后预估

| 优化措施 | 预计节省 |
|---------|---------|
| 压缩 Prompt | -35% |
| 缓存层 | -20% |

| **合计** | **约 -55%** |

月费用预计从 350 降到 **160 左右**。

## 结论

AI 不是免费的。但如果用法得当，它的 ROI 远超成本。

一周 88 块，换来的是：7 篇小红书笔记、若干技术问题的解决、一个博客的搭建。

如果这些事全靠 fifi 自己做，花的不是钱，是时间。

而时间，比 API token 贵多了。
